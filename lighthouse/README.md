# Lighthouse runs

Lighthouse JSON snapshots for tracking perf, a11y, best-practices, and SEO over time. Two flows live here side by side:

1. **Stage summaries** (committed manually) â€” `<route>-<context>.summary.json`. Hand-authored for stage-to-stage comparison, with `deltas` and `notes` blocks that tell the story behind each move. Examples: `skills-pre-stage-6.summary.json`, `skills-post-stage-13.summary.json`.
2. **Per-commit summaries** (committed by CI) â€” `<route>-<sha>.summary.json`. Auto-generated by [.github/workflows/lighthouse.yml](../.github/workflows/lighthouse.yml) after every push to `main`. Captures the same metric block; no `deltas`/`notes` (those are still hand-written when a stage moves a number meaningfully).

[RESULTS.md](RESULTS.md) is the human-readable timeline that pulls deltas from both flows.

## Automated runs (Stage 21)

The workflow:

1. Triggers on `push: main` (or manual `workflow_dispatch`).
2. Sleeps 90s to let Cloudflare Workers finish deploying the merged commit (via the `deploy.yml` workflow that also runs on push to main). The deploy doesn't expose the commit SHA in response headers, so polling-until-match isn't trivial â€” a fixed sleep is the pragmatic compromise. If you see scores capture the _previous_ deploy occasionally, bump the sleep in the workflow.
3. Runs Lighthouse against five routes: `/`, `/skills`, `/education`, `/skills/1`, `/education/degree`. Mobile profile, Lantern simulation (matching the historical baselines).
4. Writes a small summary per route to `lighthouse/<route>-<sha>.summary.json` and commits it back to `main` with `[skip ci]`.
5. Uploads the full reports as a 14-day artifact for debugging when a score moves unexpectedly.

It's **warn-only** â€” the workflow always exits 0 if Lighthouse runs successfully. To diagnose a regression, diff the latest commit's summaries against the previous one.

## Hand-authored stage summaries

When a stage merges and meaningfully moves a metric, write a `<route>-post-stage-N.summary.json` next to the auto-generated ones. Same shape, plus optional `deltas` and `notes` blocks comparing to the prior stage. Update [RESULTS.md](RESULTS.md) so the timeline stays current.

Convention for hand-authored runs:

```text
<route>-<context>.summary.json
```

- `route`: `home`, `skills`, `education`, `skills-detail`, `education-detail`.
- `context`: stage marker (`pre-stage-6`, `post-stage-6`) or descriptor (`baseline`, `react-19-trial`).

## Capture a one-off run locally

Don't need this for the regular flow (CI handles it), but if you want to score a draft branch against prod or a preview URL:

```sh
npm install --no-save lighthouse@13 chrome-launcher@1
LIGHTHOUSE_SHA=preview node scripts/run-lighthouse.mjs
```

Output lands in `lighthouse/<route>-preview.summary.json` â€” clean up before pushing if it's noise.

To score a single route via the official CLI (Chrome must be installed):

```sh
npx lighthouse https://gonzalo-alvarez-campos-cv.com/skills \
  --output=json \
  --output-path=lighthouse/skills-snapshot.json \
  --form-factor=mobile \
  --throttling-method=simulate
```

## What to compare

Diff the metric block between runs. The fields that matter:

| Field path                 | What it means           | Threshold    |
| -------------------------- | ----------------------- | ------------ |
| `metrics.fcp.numericValue` | FCP (ms)                | <1800 = good |
| `metrics.lcp.numericValue` | LCP (ms)                | <2500 = good |
| `metrics.si.numericValue`  | Speed Index (ms)        | <3387 = good |
| `metrics.tbt.numericValue` | TBT (ms)                | <200 = good  |
| `metrics.cls.numericValue` | CLS (unitless)          | <0.1 = good  |
| `categories.performance`   | Overall perf score, 0â€“1 | â‰Ą0.9 = green |
| `categories.accessibility` | A11y score              | â‰Ą0.9 = green |
| `categories.seo`           | SEO score               | â‰Ą0.9 = green |

Quick diff with `jq`:

```sh
jq '{
  perf: .categories.performance,
  fcp: .metrics.fcp.displayValue,
  lcp: .metrics.lcp.displayValue,
  si: .metrics.si.displayValue,
  failing: [.failingAudits[].id],
}' lighthouse/skills-index-abc1234.summary.json
```
